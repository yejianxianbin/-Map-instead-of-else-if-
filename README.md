判断条件时经常会写 else if语句或者是switch语句，至于这两个效率方面网上都说不准。</br>
1.Java的性能测试受环境的影响很大。</br>

考虑到用map模仿判断语句，map的key代表需要判断的条件，而value则是满足条件时执行功能。</br>
这种map用法的弊端：Java的回收机制回收不到，数据多的时候内存会扛不住。</br>

GC不会回收这两个垃圾，这个跟Java中的HashMap默认是强引用有关的。</br>

1.如果有一个值，对应的键不再使用他了，但由于key与value之间存在强引用，是不会被垃圾回收的.</br> 

2.垃圾回收器跟踪活动的对象，只要映射对象是活动的，其中的所有桶也是活动的，它们不能被回收.</br>

3.这是一个简单的demo，没有考虑条件太多的情况，大家可以进行代码优化。
